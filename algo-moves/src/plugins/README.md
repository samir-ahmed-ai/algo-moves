# Writing a `ProblemPlugin`

Every problem in Algo Moves is a **plugin**. The shell (canvas, modes, nodes, edges, inspector,
metrics, code/copy) is 100% generic — it never hardcodes a problem. You implement the contract in
`src/core/types.ts` (`ProblemPlugin`) and the shell renders it.

Set `meta.source` to the canonical reference (LeetCode URL, `HackerRank`, or `Educational`).
Imported progress problems carry a `leetcode` field in `imported/manifest.ts`; prep problems
record links in their embedded notes. See [`ATTRIBUTIONS.md`](../../ATTRIBUTIONS.md)
for platform attribution policy.

## The contract

```ts
import { definePlugin } from '../../core/types';
import { wireTeachingStack } from '../_shared/pluginKit';
import { goodCases, badCases } from './cases';
import { quiz, codePieces } from './practice';

const inputs = [/* SampleInput<I>[] */];
const verdict = (frames) => ({ ok: true, label: 'done' });

const teaching = wireTeachingStack({
  record, View, inputs, verdict,
  practice: {
    quiz,
    codePieces,
    cases: { good: goodCases, bad: badCases, intro: '…' },
    simulateQuestion: 'Which move comes next?',
  },
});

export const myPlugin = definePlugin<Input, State>({
  meta: { id: 'two-sum', title: 'Two Sum', difficulty: 'Easy', tags: ['array'], summary: '…', source: '…' },
  inputs,
  record,
  View,
  verdict,
  code: { text: solutionSource, lang: 'go', file: 'solution.go' },
  codePieces,
  quiz,
  tabs: teaching.tabs,
  wires: teaching.wires,
  extraCode: [{ text: pySource, lang: 'python', file: 'solution.py' }],
  editable: [{ key: 'values', label: 'Array', type: 'numberArray', min: 0, max: 99 }],
});
```

`wireTeachingStack` (in `_shared/pluginKit.ts`) registers Cases / Quiz / Simulate tabs and wires them
to shell built-ins. **Code Studio** reads the same `quiz` and `codePieces` for its quiz → reassemble → recall phases.

The **Practice tab quiz** and **Code Studio quiz phase** share one data source (`plugin.quiz` / `practice.quiz`) but serve different UX: the tab is inline multiple-choice with immediate feedback; Code Studio runs the same questions as a gated phase before reassemble/recall. Edit `practice.ts` (or imported `practice/items/<id>.ts`) once — both surfaces stay in sync.

**Quiz label format:** use `headline — short detail` on every choice (e.g. `O(n!) — branching narrows each deeper row`). See [`docs/quiz-and-code-studio.md`](../../docs/quiz-and-code-studio.md) for rendering, shuffle, restart-on-wrong, and integrity rules.

## Shared teaching factories

| Module | Role |
|--------|------|
| `_shared/practice.tsx` | `makeCasesPanel`, `makeQuizPanel`, `makeSimulatePanel` |
| `_shared/pluginKit.ts` | `wireTeachingStack`, `PracticeBundle`, `codePiecesFromSource` |
| `_shared/vizKit.tsx` | `vizText`, `VizHint`, `PathDisplay`, `CharCell`, `DpCell`, `makeInspector` |
| `_shared/vizTokens.ts` | Typography tokens (`--node-fs*` / `--viz-*`) for plugin views |
| `_shared/inspector.tsx` | Re-exports `InspectorRow` / `VarGrid` from vizKit |
| `_shared/graphLayout.ts` | `circleLayout`, `gridLayout`, `layeredLayout` |
| `_shared/sortRecorder.ts` | `createSortRecorder`, selection/insertion helpers, `sortVerdict` |

**Imported library** problems use `makeImportedPlugin()` in `imported/factory.tsx`, which auto-attaches
the learn stack when a simulator + practice bundle exist. Curated duplicates (e.g. climbing-stairs) point
at imported `pluginId`s — see `imported/practice/migrated.ts`.

**Prep library** is a second import path: `prepManifest.ts` (generated by `npm run import-prep`) →
`makePrepPlugin()` in `imported/prepFactory.tsx`. Every current prep
problem has a bespoke step-simulator in `imported/prepSimulators/problems/<slug>.tsx` (keyed by
manifest id, e.g. `prep-design-lru-cache`). `prepScene.tsx` is the animated Scene fallback for future
imports that do not yet have a simulator.

To add a prep simulator: `npm run scaffold-prep-sim -- <slug-or-manifestId>`, implement
`record` / `View` / `Inspector`, then run `npm run check-prep-sim-coverage` (also in `check:all`).

## Quickstart

`npm run new-problem -- two-sum "Two Sum"` scaffolds `index.tsx`, `cases.ts`, and `practice.ts`.
`npm test` runs recorder + integrity tests and `check-orphans`.

**Worked example:** [`EXAMPLE.md`](./EXAMPLE.md) — native binary search + imported word search end-to-end.

## Wires

```ts
wires: {
  visualize: [['problem', 'cases']],
  practice: [['quiz', 'simulate'], ['simulate', 'predict']],
}
```

## Register it

1. Append to `src/plugins/index.ts`
2. Add `{ id: '…', kind: 'problem', pluginId: '…' }` in `src/content/courses.ts`

Reference: `src/plugins/binary-search/` (hand-crafted) or any imported simulator under `imported/simulators/problems/`.

## Viz kit (typography + layout)

Plugin **View** and **Inspector** bodies should import from `_shared/vizKit.tsx`, not hardcode `text-[Npx]`. Viz boards render inside `.algo-canvas` nodes and inherit `--node-fs*` / `--viz-*` from `theme.css`.

```tsx
import { VizHint, VizStatRow, VizInspector, PathDisplay } from '../_shared/vizKit';

function View({ frame }: PluginViewProps<State>) {
  return (
    <VizBoard>
      <VizHint>…</VizHint>
      <PathDisplay value={frame.state.path} />
    </VizBoard>
  );
}

function Inspector({ frame }: InspectorProps<State>) {
  if (!frame) return <VizEmpty />;
  return (
    <VizInspector>
      <VizStatRow k="n" v={frame.state.n} />
    </VizInspector>
  );
}
```

Run `node scripts/lint-plugin-typography.mjs` before committing plugin UI changes.
