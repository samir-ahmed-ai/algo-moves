## Problem sources

Problems in the library draw from well-known interview platforms and original teaching exercises. Platform names and problem titles are used **only to identify the study material** — the same way a textbook cites its exercise sources.

| Source | How it appears | What is original in this repo |
|--------|----------------|-------------------------------|
| [**LeetCode**](https://leetcode.com/) | ~100 progress-library problems link out via **Open on LeetCode ↗**; many prep-library titles match LeetCode classics | Go solutions, step simulators, move transcripts, quizzes, Code Studio exercises, SPARK mnemonic scenes |
| [**HackerRank**](https://www.hackerrank.com/) | Five problems tagged `(HackerRank)` — BFS Shortest Reach, Floyd: City of Blinding Lights, Merging Communities, Subset Component, Array Manipulation | Same as above — all implementation and visualization code is authored here |
| **Educational** | Pattern-first scaffolding (unweighted BFS, graph traversal, topological sort, reachability) marked `Educational` in manifests | Problem framing, solutions, and simulators |
| **Original** | 18 native plugins (binary search, six sorting algorithms, n-queens, tree traversals, trie, union-find, …) plus bespoke prep simulators | Entirely original |

Nothing in this repository redistributes LeetCode or HackerRank problem statements, test harnesses, or official editorial solutions.
